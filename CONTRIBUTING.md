# Contributing to QML JavaScript IntelliSense

Thank you for your interest in contributing to QML JavaScript IntelliSense! 

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/qml-js-intellisense.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes by pressing F5 in VS Code to launch the Extension Development Host
6. Commit your changes: `git commit -am 'Add some feature'`
7. Push to the branch: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Setup

### Prerequisites
- Node.js (v14 or higher)
- VS Code (v1.80.0 or higher)

### Install Dependencies
```bash
npm install
```

### Testing
To test the extension:
1. Open the project in VS Code
2. Press `F5` to launch the Extension Development Host
3. Open a QML project with JavaScript imports
4. Test the IntelliSense features

## Code Style

- Use 4 spaces for indentation
- Use single quotes for strings
- Add JSDoc comments for all functions
- Follow the existing code structure
- Keep functions focused and simple

## Reporting Issues

When reporting issues, please include:
- VS Code version
- Extension version
- Sample QML and JS files that reproduce the issue
- Expected vs actual behavior
- Any error messages from the Developer Tools Console

## Feature Requests

Feature requests are welcome! Please open an issue describing:
- The use case for the feature
- How it would work
- Any relevant examples

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help create a welcoming environment for all contributors
